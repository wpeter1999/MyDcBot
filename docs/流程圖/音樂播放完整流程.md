# 音樂播放完整流程

> 從使用者執行指令到音樂開始播放的完整流程

## 播放指令完整流程

```mermaid
flowchart TD
    Start([使用者執行 /play]) --> Defer[延遲回應]
    Defer --> GetQuery[取得查詢參數]
    
    GetQuery --> ValidateService{驗證服務}
    ValidateService -->|失敗| E1[❌ 服務未初始化]
    ValidateService -->|成功| ResolveYT[YouTube 解析]
    
    ResolveYT --> Timeout[設定逾時 30 秒]
    Timeout --> CallResolver[呼叫 Resolver.Resolve]
    
    CallResolver --> YtDlp[執行 yt-dlp -j]
    YtDlp -->|失敗| E2[❌ 解析失敗]
    YtDlp -->|成功| ParseJSON[解析 JSON]
    
    ParseJSON --> CreateSong[建立 Song 結構]
    CreateSong --> CheckVoice{檢查語音頻道}
    
    CheckVoice -->|未加入| E3[❌ 必須先加入語音頻道]
    CheckVoice -->|已加入| GetContext[取得 guildID & channelID]
    
    GetContext --> CheckPlaylist{是否為播放清單?}
    CheckPlaylist -->|是| PlaylistFlow[播放清單流程]
    CheckPlaylist -->|否| GetPlayer[取得 GuildPlayer]
    
    GetPlayer --> CheckPlaying{是否正在播放?}
    CheckPlaying -->|否| Dequeue[從佇列取出並播放]
    CheckPlaying -->|是| EnqueueOnly[只加入佇列]
    
    Dequeue --> SetCurrent[設定為 CurrentSong]
    SetCurrent --> JoinVoice[加入語音頻道]
    
    JoinVoice --> LoadTrack[Lavalink LoadTrack]
    LoadTrack -->|失敗| TryFallback[嘗試 SoundCloud]
    LoadTrack -->|成功| PlayTrack[Lavalink 播放]
    
    TryFallback -->|失敗| E4[❌ 播放失敗]
    TryFallback -->|成功| PlayTrack
    
    PlayTrack --> UpdateStatus[更新頻道狀態]
    UpdateStatus --> ResponsePlay[回應: 正在播放]
    
    EnqueueOnly --> ResponseQueue[回應: 已加入佇列]
    
    ResponsePlay --> Success([✅ 完成])
    ResponseQueue --> Success
    
    style Start fill:#5865F2
    style Success fill:#57F287
    style E1 fill:#ED4245
    style E2 fill:#ED4245
    style E3 fill:#ED4245
    style E4 fill:#ED4245
```

## YouTube 解析流程

```mermaid
flowchart TD
    Start([查詢字串]) --> CheckType{檢查類型}
    
    CheckType -->|http:// 或 https://| IsURL[判定為 URL]
    CheckType -->|其他| IsSearch[判定為搜尋]
    
    IsURL --> CleanURL[清理 URL<br/>移除 &list= 等參數]
    IsSearch --> AddPrefix[加入 ytsearch1: 前綴]
    
    CleanURL --> BuildArgs[建構 yt-dlp 參數]
    AddPrefix --> BuildArgs
    
    BuildArgs --> Args1[-j 輸出 JSON]
    BuildArgs --> Args2[--no-warnings]
    BuildArgs --> Args3[--no-playlist]
    
    Args1 --> Execute[執行 yt-dlp]
    Args2 --> Execute
    Args3 --> Execute
    
    Execute --> CheckOutput{執行結果}
    CheckOutput -->|失敗| Error[回傳錯誤]
    CheckOutput -->|成功| ParseJSON[解析 JSON]
    
    ParseJSON --> ExtractFields[提取欄位]
    ExtractFields --> Title[title]
    ExtractFields --> WebpageURL[webpage_url]
    ExtractFields --> StreamURL[url 串流位址]
    
    Title --> CreateSong[建立 Song]
    WebpageURL --> CreateSong
    StreamURL --> CreateSong
    
    CreateSong --> SetRequester[設定 RequestedBy]
    SetRequester --> Return([回傳 Song])
    
    style Start fill:#5865F2
    style Return fill:#57F287
    style Error fill:#ED4245
```

## 語音連接流程

```mermaid
sequenceDiagram
    participant Bot
    participant Discord as Discord 閘道
    participant Lavalink
    participant Voice as Discord 語音
    
    Bot->>Discord: UpdateVoiceState(channelID)
    Note over Discord: 請求加入語音頻道
    
    Discord->>Bot: VoiceStateUpdate 事件
    Note over Bot: sessionID
    
    Discord->>Bot: VoiceServerUpdate 事件
    Note over Bot: token, endpoint
    
    Bot->>Lavalink: OnVoiceStateUpdate(guildID, channelID, sessionID)
    Bot->>Lavalink: OnVoiceServerUpdate(guildID, token, endpoint)
    
    Lavalink->>Voice: 建立 WebSocket 連線
    Note over Voice: 使用 token 和 endpoint
    
    Voice-->>Lavalink: 連線成功
    Lavalink-->>Bot: 準備就緒
    
    Bot->>Lavalink: LoadTrack(url)
    Lavalink-->>Bot: Track 載入完成
    
    Bot->>Lavalink: Player.Update(WithTrack)
    Lavalink->>Voice: 開始傳送音訊
    
    Voice-->>Bot: 播放中
```

## Lavalink LoadTrack 流程

```mermaid
flowchart TD
    Start([LoadTrack 請求]) --> GetNode[取得最佳節點<br/>BestNode]
    GetNode --> Handler[設定 ResultHandler]
    
    Handler --> Send[發送 LoadTrack 請求]
    Send --> Wait[等待 Lavalink 回應]
    
    Wait --> CheckType{回應類型}
    
    CheckType -->|track| Single[單個音軌]
    CheckType -->|playlist| Playlist[播放清單]
    CheckType -->|search| Search[搜尋結果]
    CheckType -->|empty| NoMatch[無匹配]
    CheckType -->|error| LoadError[載入錯誤]
    
    Single --> UseTrack[使用該音軌]
    Playlist --> UseFirst[使用第一首]
    Search --> UseFirst
    NoMatch --> Error1[回傳錯誤: 無匹配]
    LoadError --> Error2[回傳錯誤: 載入失敗]
    
    UseTrack --> PlayerUpdate[Player.Update]
    UseFirst --> PlayerUpdate
    
    PlayerUpdate --> StartPlay[開始播放]
    StartPlay --> Success([播放中])
    
    style Start fill:#5865F2
    style Success fill:#57F287
    style Error1 fill:#ED4245
    style Error2 fill:#ED4245
```

## 播放失敗備用流程

```mermaid
flowchart TD
    Start([嘗試播放]) --> TryYtDlp[JoinVoiceAndPlayWithYtDlp]
    
    TryYtDlp --> CheckYtDlp{成功?}
    CheckYtDlp -->|是| Success1[播放成功]
    CheckYtDlp -->|否| LogYtDlp[記錄: yt-dlp 失敗]
    
    LogYtDlp --> BuildSC[建構 SoundCloud 查詢<br/>scsearch: + 標題]
    BuildSC --> TrySC[JoinVoiceAndPlay with SC]
    
    TrySC --> CheckSC{成功?}
    CheckSC -->|是| Success2[播放成功 via SC]
    CheckSC -->|否| LogSC[記錄: SoundCloud 失敗]
    
    LogSC --> Failed[播放失敗]
    
    Success1 --> UpdateStatus[更新頻道狀態]
    Success2 --> UpdateStatus
    UpdateStatus --> Complete([完成])
    
    Failed --> Error([回傳錯誤])
    
    style Start fill:#5865F2
    style Complete fill:#57F287
    style Error fill:#ED4245
```

## 佇列處理邏輯

```mermaid
flowchart TD
    Start([播放請求]) --> GetPlayer[取得 GuildPlayer]
    GetPlayer --> Enqueue[Enqueue 歌曲]
    
    Enqueue --> CheckFull{佇列已滿?}
    CheckFull -->|是| Error[❌ ErrQueueFull]
    CheckFull -->|否| Added[加入成功]
    
    Added --> CheckState{播放器狀態}
    
    CheckState -->|正在播放| JustQueued[只加入佇列<br/>回應: 已加入佇列]
    CheckState -->|閒置| NeedPlay[需要開始播放]
    
    NeedPlay --> Dequeue[Dequeue 取出]
    Dequeue --> SetCurrent[SetCurrentSong]
    SetCurrent --> StartPlay[開始播放]
    
    StartPlay --> Success1[回應: 正在播放]
    JustQueued --> Success2[回應: 已加入佇列]
    
    Success1 --> End([完成])
    Success2 --> End
    
    style Start fill:#5865F2
    style End fill:#57F287
    style Error fill:#ED4245
```

## 頻道狀態更新流程

```mermaid
flowchart TD
    Start([歌曲開始播放]) --> Format[格式化狀態<br/>🎵 + 歌曲標題]
    Format --> CheckLength{長度檢查}
    
    CheckLength -->|> 500| Truncate[截斷為 497 字元 + ...]
    CheckLength -->|<= 500| BuildPayload[建構請求 payload]
    
    Truncate --> BuildPayload
    BuildPayload --> CreateRequest[建立 HTTP PUT 請求]
    
    CreateRequest --> SetHeaders[設定標頭]
    SetHeaders --> Auth[Authorization: Bot TOKEN]
    SetHeaders --> ContentType[Content-Type: application/json]
    
    Auth --> Send[發送到 Discord API]
    ContentType --> Send
    
    Send --> CheckResponse{回應狀態碼}
    
    CheckResponse -->|204 或 200| Success[更新成功]
    CheckResponse -->|其他| LogError[記錄錯誤]
    
    Success --> Complete([完成])
    LogError --> Complete
    
    style Start fill:#5865F2
    style Success fill:#57F287
    style LogError fill:#FEE75C
```

## 播放器狀態機

```mermaid
stateDiagram-v2
    [*] --> Idle: 建立播放器
    
    Idle --> Loading: 加入第一首歌
    Loading --> Connecting: 解析成功
    Connecting --> Playing: 連線語音成功
    
    Playing --> Playing: 正常播放中
    Playing --> Paused: 執行 /pause
    Paused --> Playing: 再次執行 /pause
    
    Playing --> Loading: 執行 /skip
    Loading --> Playing: 下一首載入成功
    Loading --> Idle: 佇列為空
    
    Playing --> Stopped: 執行 /stop
    Paused --> Stopped: 執行 /stop
    Stopped --> [*]: 清理完成
    
    Loading --> Error: 載入失敗
    Connecting --> Error: 連線失敗
    Error --> Loading: 重試下一首
    Error --> Idle: 無更多歌曲
```

## 錯誤處理決策

```mermaid
flowchart TD
    Start([發生錯誤]) --> Type{錯誤類型}
    
    Type -->|服務未初始化| E1[初始化檢查失敗]
    Type -->|YouTube 解析失敗| E2[URL 無效或網路問題]
    Type -->|未在語音頻道| E3[使用者狀態問題]
    Type -->|佇列已滿| E4[容量限制]
    Type -->|Lavalink 連線失敗| E5[服務問題]
    Type -->|音軌載入失敗| E6[來源問題]
    
    E1 --> Log[記錄錯誤]
    E2 --> Log
    E3 --> Log
    E4 --> Log
    E5 --> Log
    E6 --> TryFallback{有備用方案?}
    
    TryFallback -->|是| Fallback[嘗試 SoundCloud]
    TryFallback -->|否| Log
    
    Fallback -->|成功| Success[播放成功]
    Fallback -->|失敗| Log
    
    Log --> Response[回應使用者]
    Success --> Response
    
    style Start fill:#FEE75C
    style Success fill:#57F287
    style Response fill:#5865F2
```
