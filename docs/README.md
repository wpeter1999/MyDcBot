# 使用說明

> 如何使用 Discord 音樂機器人 Obsidian 索引系統

## 📁 目錄結構

```
docs/
├── INDEX.md                    # 主索引（從這裡開始）
├── README.md                   # 本檔案
├── 功能索引.md                  # 函式快速查找
├── 功能模組/                    # 按功能分類的文件
│   ├── 音樂播放功能.md
│   ├── 播放控制功能.md
│   ├── 佇列管理功能.md
│   ├── 下載功能.md
│   ├── 播放清單功能.md
│   ├── Lavalink整合.md
│   └── Bot核心.md
├── 流程圖/                      # 核心流程的詳細圖表
│   └── 自動播放下一首流程.md
└── 知識庫/                      # 專業術語和概念解釋
    └── 專有名詞索引.md
```

## 🚀 快速開始

### 1. 安裝 Obsidian

1. 下載並安裝 [Obsidian](https://obsidian.md/)
2. 啟動 Obsidian
3. 選擇 "Open folder as vault"
4. 選擇專案根目錄：`e:\web_project\DiscordBot`

### 2. 從主索引開始

開啟 `docs/INDEX.md`，這是整個索引的入口。

### 3. 使用內部連結導航

點擊 `[[文件名稱]]` 或 `[連結文字](檔案路徑.md)` 格式的連結可以跳轉到對應文件。

## 📖 文件組織方式

### 功能模組文件

每個功能模組文件包含：

1. **功能概述** - 該模組的作用和特性
2. **流程圖** - Mermaid 圖表展示核心流程
3. **核心函式** - 按函式分解，包含：
   - 位置（檔案路徑和行號）
   - 功能說明
   - 參數和回傳值
   - 程式碼範例
   - 相關函式連結
4. **錯誤處理** - 常見錯誤和解決方案
5. **測試覆蓋** - 測試檔案和覆蓋率

### 流程圖文件

詳細的端到端流程說明：
- 完整流程圖
- 狀態機圖
- 時序圖
- 步驟分解

### 知識庫文件

專業術語解釋：
- 技術概念
- 縮寫對照
- API 說明

## 🎯 常見使用場景

### 場景 1：理解某個功能如何運作

1. 開啟 `INDEX.md`
2. 在「功能模組文件」部分找到對應功能
3. 點擊連結查看詳細文件
4. 閱讀流程圖和核心函式

### 場景 2：查找特定函式的用途

1. 開啟 `功能索引.md`
2. 在表格中搜尋函式名稱
3. 點擊連結查看函式文件

### 場景 3：理解完整流程

1. 開啟 `INDEX.md`
2. 在「核心流程圖」部分找到對應流程
3. 查看詳細的流程圖和步驟說明

### 場景 4：查詢專業術語

1. 開啟 `知識庫/專有名詞索引.md`
2. 按字母順序查找術語
3. 閱讀解釋和相關文件連結

## 🔍 搜尋技巧

### 全域搜尋（Obsidian）

- **快速鍵**：`Ctrl+Shift+F`（Windows）或 `Cmd+Shift+F`（Mac）
- **用途**：在所有文件中搜尋關鍵字

### 快速切換（Obsidian）

- **快速鍵**：`Ctrl+O`（Windows）或 `Cmd+O`（Mac）
- **用途**：快速開啟文件

### 文字編輯器搜尋

也可以直接用 VS Code 或其他編輯器開啟 `docs/` 目錄進行搜尋。

## 📊 Mermaid 圖表說明

所有流程圖使用 Mermaid 語法，Obsidian 原生支援渲染。

### 流程圖類型

1. **Flowchart** - 基本流程圖
2. **Sequence Diagram** - 時序圖
3. **State Diagram** - 狀態機圖
4. **Class Diagram** - 類別圖
